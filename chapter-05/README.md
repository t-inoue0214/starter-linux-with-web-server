# 第5章: テキストエディタ3種を使う

## 前提知識

この章を始める前に、以下の章を完了していること:

- 第2章: 基本コマンドを使いこなす
- 第4章: パッケージ管理（nginx がインストール済みであること — 5-6 で使用）

---

## 概要

Linux サーバーには GUI（グラフィカルな画面）がありません。
設定ファイルを変更するときは、ターミナル上で動く「テキストエディタ」を使います。
この章では代表的な3種（`vim` / `nano` / `emacs`）の使い方を学びます。

第4章で「謎」として残した `/etc/nginx/nginx.conf` を、この章の最後で実際に `vim` で開きます。
第17章でソースビルド後の nginx 設定を編集するとき、今日学ぶ操作が必要になります。

---

## 手順

### 5-1. なぜターミナルエディタを学ぶのか

#### サーバーに GUI がない理由

本番の Linux サーバーは、通常 SSH（**SSH**: Secure Shell — 暗号化された経路でリモートのサーバーを操作する仕組み。第12章で詳しく学びます）でリモート接続して操作します。
GUI ソフト（VS Code 等）でファイルを開くことはできず、ターミナル上のエディタが唯一の手段です。
Codespaces も内部は Linux コンテナであり、`sudo` 権限が必要なファイルは VS Code の GUI から直接書き込めません。

```bash
# VS Code の GUI ではこのファイルを保存できない（root 所有のため）
$ sudo vim /etc/nginx/nginx.conf    # ターミナルエディタなら sudo 付きで開ける
```

#### 3種の特徴比較

| エディタ | 特徴 | 学習コスト | インストール |
|:---|:---|:---|:---|
| `vim` | モード切替型・高速操作・ほぼ全 Linux に標準搭載 | 高 | 通常プリインストール |
| `nano` | 画面下部に操作ガイドを常時表示・初心者向け | 低 | 通常プリインストール |
| `emacs` | 拡張性の高い独自エコシステム | 高 | 別途インストールが多い |

---

### 5-2. vim を使う

vim の最大の特徴は**モード**の概念です。
テキストを入力する「インサートモード」と、コマンドを受け付ける「ノーマルモード」を切り替えて操作します。
初めてだと「何もできない」と感じますが、モードさえ理解すれば強力なエディタになります。

#### モードの切り替え

```text
起動直後 → ノーマルモード（カーソル移動・コマンド）
           │
           i（または a、o）
           ↓
         インサートモード（テキスト入力）
           │
           Esc
           ↓
         ノーマルモード（戻る）
```

> **困ったら `Esc` を押す**
> どのモードにいるかわからなくなったら、まず `Esc` を押してノーマルモードに戻りましょう。

#### 実習: 練習ファイルを vim で編集する

```bash
# 練習用ファイルを作成して開く
$ vim ~/practice.txt
```

ファイルが開いたら、以下の操作を順番に試してください:

```text
1. i を押す          → インサートモードに切り替わる（画面下に -- INSERT -- と表示）
2. 「Hello, vim!」と入力
3. Esc を押す        → ノーマルモードに戻る
4. :w を入力して Enter → 保存
5. :q を入力して Enter → 終了
```

#### vim 主要操作一覧

| 操作 | コマンド |
|:---|:---|
| インサートモードへ（カーソル位置） | `i` |
| インサートモードへ（カーソルの後） | `a` |
| インサートモードへ（次の行を挿入） | `o` |
| ノーマルモードへ | `Esc` |
| 保存 | `:w` |
| 終了 | `:q` |
| 保存して終了 | `:wq` または `ZZ` |
| 保存せず強制終了 | `:q!` |
| 検索 | `/検索語`（次: `n`、前: `N`） |
| 行削除 | `dd` |
| 行コピー | `yy` |
| 貼り付け | `p` |
| 元に戻す | `u` |
| やり直し | `Ctrl+r` |
| 一括置換 | `:%s/old/new/g` |
| 行番号を表示 | `:set number` |

### 5-3. nano を使う

nano は操作方法が画面下部に常時表示されるため、初めてのターミナルエディタとして扱いやすいです。
`^` は `Ctrl` キーを意味します。

#### 実習: 先ほどの練習ファイルを nano で開く

```bash
$ nano ~/practice.txt
```

```text
1. 内容を適当に編集する
2. Ctrl+O を押す → ファイル名の確認が出る → Enter で保存確定
3. Ctrl+X を押す → 終了
```

#### nano 主要操作一覧

| 操作 | コマンド |
|:---|:---|
| 保存 | `Ctrl+O`（Write Out） |
| 終了 | `Ctrl+X` |
| 検索 | `Ctrl+W` |
| 行削除 | `Ctrl+K` |
| 削除した行を貼り付け | `Ctrl+U` |
| ヘルプ | `Ctrl+G` |

> **注意: `Ctrl+S` を押さない**
> Linux ターミナルでは `Ctrl+S` を押すと端末の出力が一時停止します（フロー制御）。
> 固まったように見えますが、`Ctrl+Q` を押せば解除できます。

---

### 5-4. emacs を使う

emacs がインストールされているか確認してみましょう。Codespaces 環境にはプリインストールされています。

```bash
$ which emacs
/usr/bin/emacs    # このように表示されればインストール済み

# 表示されない環境の場合はインストールする
$ sudo apt install emacs
```

emacs のキーバインド（**キーバインド**: キーボードショートカットの割り当てと表記法のこと）は `C-x`（Ctrl+x）や `M-x`（Alt+x、または Esc→x）という表記を使います。

#### 実習: 先ほどの練習ファイルを emacs で開く

```bash
$ emacs ~/practice.txt
```

```text
1. そのまま文字を入力できる（モード切替は不要）
2. Ctrl+X を押してから Ctrl+S を押す → 保存
3. Ctrl+X を押してから Ctrl+C を押す → 終了
   （変更があると「Save file?」と確認が出るので y を押す）
```

#### emacs 主要操作一覧

| 操作 | コマンド |
|:---|:---|
| 保存 | `C-x C-s`（Ctrl+x を押してから Ctrl+s） |
| 終了 | `C-x C-c` |
| 検索 | `C-s` |
| 行削除 | `C-k` |
| コマンド入力 | `M-x`（M = Alt または Esc） |

> **emacs から抜け出せなくなったら**
> `C-x C-c` で終了できます。変更を保存するか確認が出たら `y` を押します。

### 5-5. デフォルトエディタを設定する

Linux では `git commit` や `visudo` など、エディタが自動的に起動する場面があります。
そのときどのエディタが開くかは、`EDITOR` 環境変数とシステム設定で決まります。

#### EDITOR 環境変数（現在のシェルのみ有効）

> **「環境変数」とは?**
> OS が管理するキーと値のペアで、プログラムに設定を伝える仕組みです（Windows の「システムのプロパティ → 環境変数」に相当）。
> `export EDITOR=vim` は「EDITOR という名前の変数に vim という値をセットする」という操作です。
> 環境変数の仕組み全体は第6章で詳しく学びます。

```bash
# 現在の設定を確認
$ echo $EDITOR
（出力なし: まだ EDITOR が設定されていない場合は何も表示されません）

# vim に設定（現在のシェルセッションのみ有効）
$ export EDITOR=vim

# 確認
$ echo $EDITOR
vim
```

#### update-alternatives（システム全体のデフォルト変更）

```bash
$ sudo update-alternatives --config editor
```

```text
There are 4 choices for the alternative editor (providing /usr/bin/editor).

  Selection    Path                Priority   Status
------------------------------------------------------------
* 0            /bin/nano            40        auto mode
  1            /bin/nano            40        manual mode
  2            /usr/bin/emacs       0         manual mode
  3            /usr/bin/vim.basic   30        manual mode
  4            /usr/bin/vim.tiny    15        manual mode

Press <enter> to keep the current choice[*], or type selection number:
```

番号を入力して Enter を押すとデフォルトエディタが変わります。

> **`EDITOR` vs `update-alternatives` の違い**
> `export EDITOR=vim` は現在のシェルセッションのみ有効（ログアウトすると元に戻る）。
> `update-alternatives` はシステム全体に永続的に反映される。

### 5-6. nginx.conf を vim で開く（謎リスト問1に回答）

第4章の謎リスト: 「`/etc/nginx/nginx.conf` はどうやって編集する?」

答えは `sudo vim /etc/nginx/nginx.conf` です。
今回は誤って設定を壊さないよう、**読み取り専用モード** `-R` で開きます。

```bash
# 読み取り専用で nginx.conf を開く（保存しようとしてもエラーになる）
$ sudo vim -R /etc/nginx/nginx.conf
```

開いたら以下の操作で内容を眺めてみましょう:

```text
/listen    → 「listen」を検索（80番ポートの設定が見つかる）
n          → 次の検索結果へ
:q         → 終了（読み取り専用なので :q! は不要）
```

> **今は内容を理解できなくて OK**
> `worker_processes`・`events`・`http` ブロックなど、設定の意味は第9〜15章を通じて少しずつわかってきます。
> 第19章ではソースビルド後の `/usr/local/nginx/conf/nginx.conf` を実際に編集します。

## よくあるミス

| ミス | 内容 | 対処 |
|:---|:---|:---|
| vim でモードを確認せずに入力 | ノーマルモードのまま文字を打つとコマンドとして解釈される | `Esc` を押してノーマルモードに戻る |
| インサートモードのまま `:wq` と入力 | `:wq` という文字がテキストに挿入される | `Esc` → `:wq` の順番で操作する |
| nano で `Ctrl+S` を押す | 端末出力が一時停止して固まったように見える | `Ctrl+Q` で解除する |
| emacs の終了方法がわからない | 抜け出せなくなる | `C-x C-c`（Ctrl+x を押してから Ctrl+c） |

## 類似比較

| 操作 | vim | nano | emacs |
|:---|:---|:---|:---|
| テキスト入力 | `i` キーでインサートモードに入ってから入力 | 開いてすぐ入力できる | 開いてすぐ入力できる |
| 保存して終了 | `Esc` → `:wq` | `Ctrl+X` → `Y` → Enter | `C-x C-s` → `C-x C-c` |
| 保存せず終了 | `Esc` → `:q!` | `Ctrl+X` → `N` | `C-x C-c` → `n` |
| 検索 | `/検索語`（`n` で次候補） | `Ctrl+W` | `C-s 検索語` |
| 学習コスト | 高（モード切替の概念が独特） | 低（直感的に操作できる） | 高（独自キーバインドが多い） |

## 理解度チェック

1. vim でテキストを入力するには、最初にどのキーを押す必要があるか？

<details><summary>答え</summary>

`i` を押してインサートモードに切り替えます。
起動直後は「ノーマルモード」になっているため、そのままキーを押してもコマンドとして解釈されます。

</details>

2. vim で変更を保存せずに終了するコマンドは？

<details><summary>答え</summary>

`:q!` を入力します。
`!` をつけることで変更を破棄して強制終了します。
（`!` なしの `:q` は変更がある場合、エラーになります）

</details>

3. nano で `Ctrl+S` を押すと何が起きるか？どう解除するか？

<details><summary>答え</summary>

ターミナルの出力が一時停止し、固まったように見えます（フロー制御機能）。
`Ctrl+Q` を押すと解除できます。

</details>

4. `export EDITOR=vim` と `sudo update-alternatives --config editor` の違いは？

<details><summary>答え</summary>

- `export EDITOR=vim`: 現在のシェルセッションのみ有効。ログアウトすると元に戻る。
- `update-alternatives --config editor`: システム全体に永続的に反映される。

</details>

5. `sudo vim -R /etc/nginx/nginx.conf` の `-R` オプションと `sudo` はそれぞれ何のために必要か？

<details><summary>答え</summary>

- `-R`: 読み取り専用（Read-only）モードで開く。保存しようとするとエラーになるため、誤って設定を変更するリスクがない。
- `sudo`: `/etc/nginx/nginx.conf` は root ユーザーが所有しているため一般ユーザーではそのまま開けない（第11章「パーミッション」で詳しく学ぶ）。

</details>
